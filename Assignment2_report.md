## 1. Project Overview
The objective of this assignment was to build a personal static blog and deploy it using GitHub Pages. This project demonstrates proficiency in using Git for version control and managing remote deployment workflows.

## 2. Tools & Methodology
* **Operating System:** Windows Subsystem for Linux (WSL) - Ubuntu.
* **Version Control:** Git.
* **Hosting Service:** GitHub Pages.
* **Development:** HTML5/CSS3 for simple and accessible web design.

## 3. Git Commit Rationale
I followed a structured 5-step commit process to maintain a clear project history:
1. **Initial setup**: Built the foundational HTML structure.
2. **Integration**: Imported the Markdown and PDF reports from Assignment 1.
3. **Navigation**: Created hyperlinks to ensure all documents are accessible from the homepage.
4. **Documentation**: Authored this deployment report to explain the project details.
5. **Final Polish**: Enhanced the UI/UX with internal CSS and verified all links.

## 4. Deployment Steps
1. Initialized the repository locally and managed files through the command line.
2. Created a remote repository on GitHub.
3. Enabled GitHub Pages via the repository settings, using the 'main' branch as the source.

## 5. Challenges & Solutions
During the deployment, I faced network connectivity issues with the GitHub API from my local environment. I resolved this by manually syncing and refining the commit history through the GitHub web interface to ensure the project met the versioning requirements.
